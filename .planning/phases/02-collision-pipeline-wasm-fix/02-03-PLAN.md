In demo/collision-wasm.html, the render() function (around line 371) creates an empty hit array and returns without any manifold data. Apply these changes:

Change 1 - Retrieve manifolds and populate hit[]. Replace:
```javascript
const hit = new Uint8Array(bodyCount)
// no manifold retrieval
const elapsed = performance.now() - t0
```

With:
```javascript
const manifolds = world ? [...world.manifolds] : []
const hit = new Uint8Array(bodyCount)
for (const m of manifolds) {
  if (m.bodyA < bodyCount) hit[m.bodyA] = 1
  if (m.bodyB < bodyCount) hit[m.bodyB] = 1
}
const elapsed = performance.now() - t0
```

Change 2 - Add contact point and normal visualization after the drawShape loop. After `for (let i = 0; i < bodyCount; i++) drawShape(i, hit[i])`, add:

```javascript
if (showNormals || showContacts) {
  for (const m of manifolds) {
    for (const contact of m.contacts) {
      const posAX = buffers.positionX[m.bodyA]
      const posAY = buffers.positionY[m.bodyA]
      const angle = buffers.angle[m.bodyA]
      const cosA = Math.cos(angle), sinA = Math.sin(angle)
      const wx = cosA * contact.localA.x - sinA * contact.localA.y + posAX
      const wy = sinA * contact.localA.x + cosA * contact.localA.y + posAY

      if (showContacts) {
        ctx.fillStyle = '#ffcc00'
        ctx.beginPath()
        ctx.arc(wx, wy, 4, 0, Math.PI * 2)
        ctx.fill()
      }
      if (showNormals) {
        const len = 30
        ctx.strokeStyle = 'rgba(255,100,100,0.7)'
        ctx.lineWidth = 2
        ctx.beginPath()
        ctx.moveTo(wx, wy)
        ctx.lineTo(wx + m.normal.x * len, wy + m.normal.y * len)
        ctx.stroke()
        const ax = wx + m.normal.x * len
        const ay = wy + m.normal.y * len
        const perpX = -m.normal.y, perpY = m.normal.x
        ctx.beginPath()
        ctx.moveTo(ax, ay)
        ctx.lineTo(ax - m.normal.x * 6 + perpX * 4, ay - m.normal.y * 6 + perpY * 4)
        ctx.moveTo(ax, ay)
        ctx.lineTo(ax - m.normal.x * 6 - perpX * 4, ay - m.normal.y * 6 - perpY * 4)
        ctx.stroke()
      }
    }
  }
}
```

The world-space reconstruction uses cos*lx - sin*ly (standard 2D rotation), matching the mat2FromAngle convention (c*lx + negS*ly where negS = -sin).
